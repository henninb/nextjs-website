# Getting Started with NextJS and TypeScript

NextJS and TypeScript create a powerful combination for building modern web applications. This comprehensive guide will walk you through setting up a production-ready project and introduce you to the key features that make this stack so compelling for developers.

## Why Choose NextJS and TypeScript?

### NextJS: The React Production Framework

NextJS has become the de facto standard for production React applications, and for good reason:

**üöÄ Performance Out of the Box**
- **Automatic Code Splitting**: Only load JavaScript needed for each page
- **Server-Side Rendering (SSR)**: Improve SEO and initial load times
- **Static Site Generation (SSG)**: Pre-render pages at build time for maximum speed
- **Image Optimization**: Automatic image resizing, lazy loading, and modern formats

**üìÅ File-Based Routing**
- No complex routing configuration required
- Intuitive folder structure maps directly to URL structure
- Dynamic routes with bracket notation (`[id].tsx`)
- API routes alongside your pages

**üîß Built-in Optimizations**
- Automatic bundle optimization and tree shaking
- CSS and Sass support out of the box
- Built-in ESLint configuration
- Hot reloading for instant feedback

### TypeScript: JavaScript That Scales

TypeScript transforms JavaScript development by adding static type checking:

**‚úÖ Catch Errors Early**
- Find bugs at compile time, not runtime
- Prevent common mistakes like undefined variables
- Ensure API contracts are maintained

**üß† Enhanced Developer Experience**
- Intelligent autocomplete and IntelliSense
- Refactoring with confidence
- Self-documenting code through type annotations
- Better IDE support across all editors

**üìà Scalability**
- Manage complexity in large codebases
- Clear interfaces between components and modules
- Easier onboarding for new team members

## Project Setup: Step by Step

### Prerequisites

Before we start, ensure you have the following installed:

```bash
# Check Node.js version (16.14 or later required)
node --version

# Check npm version
npm --version
```

If you need to install Node.js, download it from [nodejs.org](https://nodejs.org/) or use a version manager like `nvm`:

```bash
# Using nvm (recommended)
nvm install 18
nvm use 18
```

### Creating Your NextJS + TypeScript Project

NextJS provides a convenient CLI tool to bootstrap new projects:

```bash
# Create a new NextJS project with TypeScript
npx create-next-app@latest my-awesome-app --typescript --tailwind --eslint --app

# Navigate to your project directory
cd my-awesome-app

# Start the development server
npm run dev
```

The flags explained:
- `--typescript`: Set up TypeScript configuration
- `--tailwind`: Include Tailwind CSS for styling
- `--eslint`: Configure ESLint for code quality
- `--app`: Use the new App Router (Next.js 13+)

Your project will be available at `http://localhost:3000`.

## Essential TypeScript Configurations

### Understanding `tsconfig.json`

The TypeScript configuration file controls how TypeScript compiles your code:

```json
{
  "compilerOptions": {
    "target": "es5",
    "lib": ["dom", "dom.iterable", "es6"],
    "allowJs": true,
    "skipLibCheck": true,
    "strict": true,
    "noEmit": true,
    "esModuleInterop": true,
    "module": "esnext",
    "moduleResolution": "bundler",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "jsx": "preserve",
    "incremental": true,
    "plugins": [
      {
        "name": "next"
      }
    ],
    "paths": {
      "@/*": ["./*"]
    }
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

**Key configurations explained:**
- `strict: true`: Enables all strict type checking options
- `noEmit: true`: NextJS handles compilation, so TypeScript just checks types
- `paths`: Allows absolute imports with `@/` prefix
- `jsx: "preserve"`: Let NextJS handle JSX transformation

### Creating API Routes with TypeScript

NextJS API routes can be strongly typed for better development experience:

```typescript
// app/api/posts/route.ts
import { NextRequest, NextResponse } from 'next/server';
import { Post, ApiResponse } from '@/types';

// Mock data - in a real app, this would come from a database
const mockPosts: Post[] = [
  {
    id: '1',
    title: 'Getting Started with NextJS and TypeScript',
    slug: 'getting-started-nextjs-typescript',
    content: 'Full content here...',
    excerpt: 'Learn how to build modern web applications...',
    author: {
      id: '1',
      name: 'John Doe',
      email: 'john@example.com',
      createdAt: new Date(),
      updatedAt: new Date(),
    },
    publishedAt: new Date(),
    tags: ['nextjs', 'typescript', 'react'],
    featured: true,
  },
  // ... more posts
];

export async function GET(request: NextRequest) {
  try {
    // Extract query parameters
    const { searchParams } = new URL(request.url);
    const page = parseInt(searchParams.get('page') || '1');
    const limit = parseInt(searchParams.get('limit') || '10');
    const tag = searchParams.get('tag');

    // Filter posts if tag is provided
    let filteredPosts = mockPosts;
    if (tag) {
      filteredPosts = mockPosts.filter(post => 
        post.tags.includes(tag.toLowerCase())
      );
    }

    // Pagination
    const startIndex = (page - 1) * limit;
    const endIndex = startIndex + limit;
    const paginatedPosts = filteredPosts.slice(startIndex, endIndex);

    const response: ApiResponse<{
      posts: Post[];
      pagination: {
        page: number;
        limit: number;
        total: number;
        totalPages: number;
      };
    }> = {
      data: {
        posts: paginatedPosts,
        pagination: {
          page,
          limit,
          total: filteredPosts.length,
          totalPages: Math.ceil(filteredPosts.length / limit),
        },
      },
      message: 'Posts retrieved successfully',
      success: true,
    };

    return NextResponse.json(response);
  } catch (error) {
    return NextResponse.json(
      { 
        data: null, 
        message: 'Failed to retrieve posts', 
        success: false 
      },
      { status: 500 }
    );
  }
}

export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    
    // Validate the request body
    const newPost: Omit<Post, 'id' | 'publishedAt'> = {
      title: body.title,
      slug: body.slug,
      content: body.content,
      excerpt: body.excerpt,
      author: body.author,
      tags: body.tags || [],
      featured: body.featured || false,
    };

    // In a real app, save to database
    const savedPost: Post = {
      ...newPost,
      id: Math.random().toString(36).substr(2, 9),
      publishedAt: new Date(),
    };

    const response: ApiResponse<Post> = {
      data: savedPost,
      message: 'Post created successfully',
      success: true,
    };

    return NextResponse.json(response, { status: 201 });
  } catch (error) {
    return NextResponse.json(
      { 
        data: null, 
        message: 'Failed to create post', 
        success: false 
      },
      { status: 500 }
    );
  }
}
```

### Using the API with Type Safety

Create a custom hook for fetching data with full type safety:

```typescript
// lib/hooks/usePosts.ts
import { useState, useEffect } from 'react';
import { Post, ApiResponse } from '@/types';

interface UsePostsOptions {
  page?: number;
  limit?: number;
  tag?: string;
}

interface UsePostsReturn {
  posts: Post[];
  loading: boolean;
  error: string | null;
  pagination: {
    page: number;
    limit: number;
    total: number;
    totalPages: number;
  } | null;
}

export function usePosts(options: UsePostsOptions = {}): UsePostsReturn {
  const [posts, setPosts] = useState<Post[]>([]);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);
  const [pagination, setPagination] = useState(null);

  useEffect(() => {
    async function fetchPosts() {
      try {
        setLoading(true);
        setError(null);

        const params = new URLSearchParams();
        if (options.page) params.append('page', options.page.toString());
        if (options.limit) params.append('limit', options.limit.toString());
        if (options.tag) params.append('tag', options.tag);

        const response = await fetch(`/api/posts?${params}`);
        
        if (!response.ok) {
          throw new Error('Failed to fetch posts');
        }

        const data: ApiResponse<{
          posts: Post[];
          pagination: any;
        }> = await response.json();

        if (data.success) {
          setPosts(data.data.posts);
          setPagination(data.data.pagination);
        } else {
          setError(data.message);
        }
      } catch (err) {
        setError(err instanceof Error ? err.message : 'An error occurred');
      } finally {
        setLoading(false);
      }
    }

    fetchPosts();
  }, [options.page, options.limit, options.tag]);

  return { posts, loading, error, pagination };
}
```

## Advanced Features and Best Practices

### Environment Variables and Configuration

Create environment-specific configurations:

```typescript
// lib/config.ts
interface Config {
  apiUrl: string;
  isDevelopment: boolean;
  isProduction: boolean;
  database: {
    url: string;
  };
  auth: {
    secret: string;
    providers: {
      google: {
        clientId: string;
        clientSecret: string;
      };
    };
  };
}

function createConfig(): Config {
  const requiredEnvVars = [
    'DATABASE_URL',
    'NEXTAUTH_SECRET',
    'GOOGLE_CLIENT_ID',
    'GOOGLE_CLIENT_SECRET',
  ];

  // Check for required environment variables
  requiredEnvVars.forEach((envVar) => {
    if (!process.env[envVar]) {
      throw new Error(`Missing required environment variable: ${envVar}`);
    }
  });

  return {
    apiUrl: process.env.NEXT_PUBLIC_API_URL || 'http://localhost:3000/api',
    isDevelopment: process.env.NODE_ENV === 'development',
    isProduction: process.env.NODE_ENV === 'production',
    database: {
      url: process.env.DATABASE_URL!,
    },
    auth: {
      secret: process.env.NEXTAUTH_SECRET!,
      providers: {
        google: {
          clientId: process.env.GOOGLE_CLIENT_ID!,
          clientSecret: process.env.GOOGLE_CLIENT_SECRET!,
        },
      },
    },
  };
}

export const config = createConfig();
```
